# Acceptance Checklist

Trade MVPの実機受け入れチェックリストです。スマホでWeb App URLを開き、Google DriveとGoogleスプレッドシートを同時に確認しながら実施します。

## 事前準備

- [ ] Web App URLをスマホで開ける
- [ ] `Trade_DB` スプレッドシートを確認できる
- [ ] `trades` シートにヘッダー行がある
- [ ] `Trade_Images/entry` フォルダを確認できる
- [ ] `Trade_Images/result` フォルダを確認できる
- [ ] `Config.gs` の `SPREADSHEET_ID` が正しい
- [ ] `Config.gs` の `ENTRY_FOLDER_ID` が正しい
- [ ] `Config.gs` の `RESULT_FOLDER_ID` が正しい
- [ ] `Config.gs` の `TIMEZONE` が `Asia/Tokyo` になっている
- [ ] `setupTradeSheet()` を1回実行済み
- [ ] `limit_value`、`stop_value`、`expected_reach_time` がプレーンテキスト列になっている
- [ ] スマホで利用するGoogleアカウントが画像フォルダを閲覧できる
- [ ] テスト用のエントリー画像と結果画像をスマホに用意している

## 補足テスト観点

### 入力バリデーション

- [ ] 指し値 `TP154.250` を保存できる
- [ ] 逆指し値 `SL154.000` を保存できる
- [ ] 到達予想時間 `London` を保存できる
- [ ] 到達予想時間 `20260504_2130` を保存できる
- [ ] 32文字を超える指し値、逆指し値、到達予想時間でエラーになる
- [ ] 許可外文字で分かりやすいエラーが出る
- [ ] メモは日本語、改行、記号を保存できる
- [ ] メモ1000文字超過時にエラーまたは入力制限される

### 画像アップロード

- [ ] jpgを保存できる
- [ ] jpegを保存できる
- [ ] pngを保存できる
- [ ] webpを保存できる
- [ ] HEICはMVP非推奨として扱われる
- [ ] 画像未選択時に保存処理されない
- [ ] 画像保存中はボタンが無効になる
- [ ] 保存失敗時に「画像の保存に失敗しました」が表示される

### データ整合性

- [ ] エントリー画像と結果画像が同じ `trade_id` に紐づく
- [ ] 結果画像保存で新規行が作られない
- [ ] `waiting` はホームに表示される
- [ ] `completed` はホームに表示されない
- [ ] 履歴には `waiting` と `completed` の両方が表示される
- [ ] `updated_at` が自動保存と結果保存で更新される
- [ ] 途中で画面を閉じても、次回起動時にスプレッドシートの状態が反映される
